AI 코워킹
실전 프롬프트 30종

"AI 코워킹 스타터킷"을 선택하신 여러분께 드리는 실전 프롬프트 30종이에요. 질문을 던지는 사용자에서, AI 직원에게 업무를 지시하는 관리자로 넘어가는 데 필요한 것들만 담았어요.

🎯 1인 전문가 맞춤 📋 복붙용 프롬프트 💡 제가 써보니 팁 포함 🔧 코워킹 실전 중심

"AI 코워킹 스타터킷"을 선택해주셔서 진짜 반가워요. AI와 함께 일한다는 이야기, 결국은 "AI에게 뭘 물어보느냐"에서 "AI에게 뭘 시키느냐"로 넘어가는 거였잖아요. 이 자료는 그 전환을 여러분 손에 쥐여드리기 위한 거예요.

제가 매일 쓰면서 "이건 진짜 일이 줄어든다" 싶었던 것들 위주로 담았어요. 30개를 다 외울 필요는 없어요. 지금 내가 막힌 한 가지부터 골라서 한번 해보시면, 그다음은 자연스럽게 따라오거든요. 결국은 하나씩 내 것으로 만드는 게 코워킹의 시작이에요.

— 브랜드앤피플 공감마케터 드림

A

코워킹 셋업 · 세션 관리

처음 시작할 때 가장 많이 막히는 지점이에요. 프로젝트를 어떻게 잡고, 세션을 어떻게 이어가느냐에서 결과물 퀄리티가 갈리거든요.

01

업무별 프로젝트 구조 설계받기

📍 이럴 때 — 코워킹 프로젝트를 만들긴 했는데, 폴더랑 파일을 어떻게 나눠야 할지 감이 안 잡힐 때
PROMPT 
나는 [직업/하는 일] 일을 하는 1인 전문가예요.
주로 하는 업무는 [예: 고객 상담, 강의안 제작, 콘텐츠 발행] 이렇게 3가지예요.

이 업무들을 AI 코워킹으로 효율적으로 굴리려면
프로젝트와 폴더 구조를 어떻게 잡는 게 좋을지 추천해줘.
- 어떤 단위로 프로젝트를 나눌지
- 각 프로젝트에 어떤 자료를 미리 넣어두면 좋을지
- 내가 매번 반복 설명하지 않아도 되게 하려면 뭘 세팅해둬야 할지
실제 폴더 이름까지 구체적으로 제안해줘.
💡한 끗 — "내 업무 3가지"를 구체적으로 적을수록 구조가 정확해져요. 막연하게 "마케팅"보다 "주 2회 블로그 발행"처럼 빈도까지 쓰면 좋아요.
02

긴 작업을 세션으로 끊어 설계하기

📍 이럴 때 — 큰 작업(예: 강의 커리큘럼 전체)을 한 번에 시키니 결과가 얕거나 중간에 흐트러질 때
PROMPT 
나는 [하려는 큰 작업]을 완성하고 싶어.
이걸 한 번에 다 하지 말고, 단계(세션)로 나눠서 진행하고 싶어.

먼저 전체 작업을 3~5개 단계로 쪼개서 계획을 보여줘.
각 단계가 끝나면 "다음 단계 갈까요?" 하고 나한테 확인받고 진행해줘.
지금은 1단계만 하고 멈춰. 전체 계획부터 보여줘.
💡한 끗 — "지금은 1단계만 하고 멈춰"가 진짜 핵심이에요. 이 한 줄이 없으면 AI가 끝까지 달려버려서 검토할 틈이 없거든요.
03

이전 작업 이어가기 (컨텍스트 복원)

📍 이럴 때 — 어제 하던 작업을 오늘 이어서 하고 싶은데, 처음부터 다시 설명하기 귀찮을 때
PROMPT 
지난번에 [했던 작업 / 만들던 결과물] 작업을 하다가 멈췄어.
관련 파일은 이 프로젝트 안에 있어.

먼저 그 파일들을 확인하고,
- 지금까지 어디까지 진행됐는지
- 다음에 뭘 해야 하는지
이 두 가지를 정리해서 나한테 먼저 보여줘.
내가 확인하면 그때 이어서 진행하자.
💡한 끗 — 작업 끝낼 때 "진행 상황 요약본 파일로 남겨줘"를 습관화하면, 다음날 이 프롬프트가 훨씬 잘 먹혀요.
04

나에 대한 기본 정보 한 번에 학습시키기

📍 이럴 때 — 매번 "저는 1인 코치고요, 톤은 이렇고요…"를 반복 설명하는 게 지칠 때
PROMPT 
앞으로 이 프로젝트에서 작업할 때 항상 기억할 내 기본 정보를
정리한 파일을 만들어줘. 파일 이름은 "00_나의_프로필.md"로 해줘.

내용은 이렇게 채워줘:
- 직업: [직업]
- 주요 고객: [누구를 돕는지]
- 글/말투 톤: [예: 친근한 존댓말, 격식 없음]
- 자주 쓰는 결과물: [블로그, 제안서, 강의안 등]
- 절대 쓰면 안 되는 표현: [있으면 적기]

이 파일을 기준으로 앞으로 모든 작업을 진행해줘.
💡한 끗 — 이 프로필 파일 하나가 30개 프롬프트의 토대예요. 이거 만들어두면 다른 프롬프트들 결과가 다 좋아져요.
05

작업 끝낼 때 인수인계 노트 남기기

📍 이럴 때 — 오늘 작업을 마무리하면서, 내일의 나(또는 다음 세션)에게 깔끔하게 넘기고 싶을 때
PROMPT 
오늘 작업 여기서 마무리할게.
다음에 이어서 할 수 있게 "인수인계 노트"를 파일로 만들어줘.

이렇게 정리해줘:
1. 오늘 완성한 것 (파일명 포함)
2. 아직 안 끝난 것 / 다음에 할 것
3. 다음에 시작할 때 나한테 줄 첫 프롬프트 한 줄
파일명은 "인수인계_[날짜].md"로.
💡한 끗 — 3번 "다음 첫 프롬프트 한 줄"이 진짜 꿀이에요. 내일 그거 복붙만 하면 바로 이어지거든요.
B

파일 · 문서 자동화

AI 코워킹이 일반 채팅과 가장 다른 지점이 바로 여기예요. 여러 파일을 한 번에 다루고, 반복 작업을 패턴으로 굳히는 것.

06

여러 파일 한 번에 정리·통합

📍 이럴 때 — 흩어진 메모, 자료 파일이 여러 개인데 하나로 정리된 문서가 필요할 때
PROMPT 
지금 올린 파일 [개수]개를 모두 읽고,
[목적: 예 - 강의안 한 편, 제안서 초안] 으로 통합해줘.

작업 순서:
1. 각 파일에서 핵심만 뽑아 한 줄 요약 (어떤 내용인지 먼저 보여줘)
2. 중복되는 내용은 합치고, 빠진 흐름은 채워줘
3. [원하는 형식: 마크다운 문서 / 표] 으로 최종본 만들어줘
1번까지만 먼저 하고 멈춰. 내가 확인할게.
💡한 끗 — "1번까지만 먼저"로 끊어주면, AI가 자료를 잘못 이해한 채 끝까지 가는 사고를 막을 수 있어요.
07

파일에서 원하는 정보만 추출해 표로

📍 이럴 때 — 긴 문서나 자료에서 특정 항목만 뽑아 정리표가 필요할 때 (예: 고객 리스트, 일정)
PROMPT 
올린 파일에서 다음 항목만 뽑아서 표로 정리해줘.

뽑을 항목: [예: 이름, 신청일, 문의 내용, 후속 필요 여부]
- 표는 보기 좋게 정렬해줘 ([기준: 날짜순 등])
- 정보가 없는 칸은 "미기재"로 표시
- 표 아래에 "후속 조치가 필요한 건"만 따로 목록으로 한 번 더 정리

결과는 복사해서 바로 쓸 수 있게 표로 출력해줘.
💡한 끗 — "정보 없으면 미기재로"를 안 넣으면 AI가 빈칸을 추측으로 채울 수 있거든요. 이 한 줄이 데이터 신뢰도를 지켜줘요.
08

긴 문서 → 핵심 한 장 요약본

📍 이럴 때 — 받은 자료/계약서/리포트가 너무 길어서, 핵심만 빠르게 파악해야 할 때
PROMPT 
올린 문서를 읽고, A4 한 장 분량으로 요약해줘.

이 구조로 정리해줘:
- 한 줄 핵심 (이 문서가 결국 말하는 것)
- 꼭 알아야 할 포인트 3~5개
- 내가 주의하거나 결정해야 할 것
- 모르는 용어가 있으면 그 자리에서 풀어서 설명

내가 [입장: 예 - 이걸 검토하는 사람] 입장이라는 걸 감안해서 정리해줘.
💡한 끗 — "내 입장"을 알려주면 요약의 초점이 확 달라져요. 검토자냐, 발표자냐에 따라 봐야 할 게 다르거든요.
09

같은 작업을 매번 똑같이 (작업 템플릿화)

📍 이럴 때 — 매주 반복하는 작업(주간 리포트, 정기 콘텐츠)을 매번 똑같은 품질로 뽑고 싶을 때
PROMPT 
나는 [반복 작업: 예 - 주간 업무 리포트]를 매주 만들어.
이걸 매번 똑같은 형식으로 뽑을 수 있게 "작업 지시서"를 파일로 만들어줘.

작업 지시서에 들어갈 것:
- 이 작업의 목적
- 들어가야 할 항목과 순서
- 톤·분량 기준
- 내가 매번 새로 줄 정보는 무엇인지 (입력 칸 표시)

다음부터 내가 "이 지시서대로 이번 주 거 만들어줘 + [자료]"
라고만 하면 되게 설계해줘.
💡한 끗 — 이게 진짜 AI 코워킹의 힘이에요. 한 번 지시서를 만들면, 다음부터는 자료만 주면 끝. 이게 쌓이면 카테고리 F의 "나만의 워크플로"가 돼요.
10

파일 형식 변환 + 정리 동시에

📍 이럴 때 — 메모해둔 텍스트를 깔끔한 문서로, 또는 표 데이터를 다른 형태로 바꿔야 할 때
PROMPT 
올린 [원본: 예 - 회의 메모 텍스트]를
[목표 형식: 예 - 정식 회의록 문서 / 표 / 발표 자료] 로 바꿔줘.

단순 변환만 하지 말고:
- 비어 있거나 흐름이 끊긴 부분은 자연스럽게 메워줘
- 내 톤([톤 적기])에 맞게 다듬어줘
- 결과는 바로 쓸 수 있는 완성본 파일로 만들어줘

작업 전에 "이렇게 바꿀게요" 방향을 한 줄로 먼저 알려줘.
💡한 끗 — "방향 먼저 한 줄"을 요청하면, 엉뚱한 방향으로 완성본까지 만들어버리는 낭비를 줄일 수 있어요.
C

콘텐츠 생산 가속

1인 전문가의 진짜 병목은 콘텐츠예요. 하나의 소스로 여러 채널을 채우는 게 핵심이에요.

11

음성 메모 → 블로그 글

📍 이럴 때 — 떠오른 생각을 말로 녹음/받아쓰기 해뒀는데, 글로 정리할 시간이 없을 때
PROMPT 
아래는 내가 말로 떠든 걸 받아쓴 거라 두서가 없어.
이걸 블로그 글로 정리해줘.

- 독자: [누가 읽는지]
- 톤: 친근한 존댓말, 옆자리에서 얘기하듯
- 첫 2~3줄은 "더 읽고 싶게" 만드는 도입으로
- 내 경험/생각은 살리고, 군더더기 말버릇은 정리
- 끝에 독자가 할 다음 행동 한 가지 자연스럽게 안내

[받아쓴 내용]
[여기에 붙여넣기]
💡한 끗 — 받아쓴 원문을 너무 다듬어서 넣지 마세요. 말투의 날것이 살아있을수록 글에 "사람 냄새"가 남아요.
12

글 1개 → 멀티 채널 동시 변환

📍 이럴 때 — 블로그 글 하나 썼는데, 이걸 SNS·뉴스레터로도 돌리고 싶을 때 (제일 자주 쓰게 될 거예요)
PROMPT 
올린 글(또는 아래 글)을 바탕으로, 채널별 버전을 만들어줘.
각 채널 특성에 맞게 길이·톤·구조를 다르게:

1. 페이스북용: 첫 2줄에서 "더 보기"를 누르게 만드는 훅 + 공감 유도
2. 인스타 캡션용: 짧고 리듬감 있게, 줄바꿈 살려서
3. 뉴스레터 도입부용: 받자마자 열어보고 싶은 3~4줄

원본의 핵심 메시지는 유지하되, 채널마다 새로 쓴 느낌으로.

[원본 글]
[여기에 붙여넣기 또는 파일 첨부]
💡한 끗 — 채널을 한 번에 다 시키되, 마음에 안 드는 것만 "2번만 다시"라고 콕 집어 재요청하면 시간이 확 줄어요.
13

내 톤 학습시켜서 글 쓰기

📍 이럴 때 — AI 글이 "내 말투가 아니야" 싶을 때. 내가 쓴 글을 샘플로 톤을 학습시키고 싶을 때
PROMPT 
내가 예전에 쓴 글 [2~3개]를 올릴게(또는 아래 첨부).
이 글들을 분석해서 내 글쓰기 톤의 특징을 정리해줘.

- 자주 쓰는 말투/어미
- 문장 길이와 리듬
- 독자를 부르는 방식
- 피하는 표현

정리한 다음, 이 톤 그대로 [새 주제] 글을 한 편 써줘.
"이건 내가 쓴 것 같다" 소리가 나오게.
💡한 끗 — 분석 결과를 "내_글쓰기_톤.md" 파일로 저장해두면, 앞으로 모든 글 작업에서 "이 톤 파일대로 써줘" 한 줄이면 끝나요.
14

PROMPT

[1단계 → 2단계 → 3단계 적기]

- 앞 단계 결과가 뒤 단계에 자동으로 이어지게
- 중간에 내가 수정하면 그 뒤 단계도 반영되게

💡한 끗 — 단계마다 "확인받고 넘어가기"를 꼭 넣으세요. 끊김 없이 다 돌리면 빠르긴 한데, 1단계 실수가 끝까지 따라가요. 속도보다 흐름의 안정성이 결국 더 빨라요.
